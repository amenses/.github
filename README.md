# Amenses Innovation

Welcome to the **Amenses Innovation** GitHub organization.

Amenses builds AI-first digital products and enterprise-grade SaaS platforms. This organization hosts our application code, infrastructure, design systems, and shared engineering resources.

## Engineering Principles

- **Ship with quality** — automated testing, code review, and security scanning are non-negotiable.
- **Keep it simple** — prefer clear, maintainable solutions over clever abstractions.
- **Document decisions** — significant changes are captured in ADRs, PRs, and the [engineering-standards](https://github.com/amenses-innovation/engineering-standards) repository.
- **Security by default** — least privilege, dependency hygiene, and responsible disclosure.
- **Collaborate openly** — issues, PRs, and discussions are the source of truth.

## Repository Categories

| Category | Prefix / Pattern | Description |
|----------|------------------|-------------|
| Applications | `app-*`, product names | Customer-facing and internal applications |
| Libraries | `lib-*`, `sdk-*` | Shared packages and SDKs |
| Infrastructure | `infra-*`, `terraform-*` | IaC, deployment, and platform tooling |
| Design | `design-*`, `ads-*` | Design system and brand assets |
| Documentation | `docs-*` | Public and internal documentation sites |
| Templates | `.github`, `template-*` | Organization defaults and repo templates |

See [docs/repository-naming.md](docs/repository-naming.md) for full naming conventions.

## Getting Started

1. Read the [Onboarding Guide](docs/onboarding.md).
2. Review [Contributing Guidelines](CONTRIBUTING.md).
3. Familiarize yourself with [engineering-standards](https://github.com/amenses-innovation/engineering-standards) policies.
4. Join the relevant GitHub teams (see [docs/github-teams.md](docs/github-teams.md)).

## Contribution Process

1. Open an issue or pick up an existing one.
2. Create a branch following our [branching strategy](docs/branching-strategy.md).
3. Open a pull request using the appropriate [PR template](.github/PULL_REQUEST_TEMPLATE/).
4. Ensure CI passes and obtain required reviews.
5. Merge according to the [release process](docs/release-process.md).
