1. Read the [Onboarding Guide](docs/onboarding.md)

## Contribution Process

1. Open or pick up an issue
2. Branch per [branching strategy](docs/branching-strategy.md)
3. Open a PR using the [appropriate template](.github/PULL_REQUEST_TEMPLATE/)
4. Ensure CI passes and obtain required reviews
5. Merge per [release process](docs/release-process.md)
