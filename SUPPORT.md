# Support

## Getting Help

### Engineering & Development

| Channel | Use For |
|---------|---------|
| GitHub Issues | Bugs, features, tasks, documentation requests |
| Team Slack channels | Day-to-day questions and coordination |
| Pull requests | Code-specific discussion |

Use the appropriate issue template:

- [Bug Report](.github/ISSUE_TEMPLATE/bug_report.yml)
- [Feature Request](.github/ISSUE_TEMPLATE/feature_request.yml)
- [Development Task](.github/ISSUE_TEMPLATE/task.yml)
- [Documentation Request](.github/ISSUE_TEMPLATE/documentation.yml)

### Security

For security vulnerabilities, see [SECURITY.md](SECURITY.md). **Do not** use public issue trackers for security reports.

### Access & Permissions

- Repository access: contact your engineering manager or the `@devops` team.
- GitHub team membership: see [docs/github-teams.md](docs/github-teams.md).

## Response Expectations

| Priority | Target Response |
|----------|-----------------|
| Production outage | Immediate (on-call) |
| Security vulnerability | Within 48 hours |
| Bug (high) | Within 2 business days |
| Bug (normal) | Within 5 business days |
| Feature requests | Triaged in sprint planning |

## Escalation

If you are not receiving a timely response:

1. Tag the relevant CODEOWNERS in your issue or PR.
2. Escalate to your team lead.
3. For cross-team blockers, open a Development Task issue with the `escalation` label.
