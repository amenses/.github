# Contributing to Amenses Innovation

Thank you for contributing to Amenses. This document outlines how to participate in our repositories.

## Before You Start

- Read the [Code of Conduct](CODE_OF_CONDUCT.md).
- Review policies in the [engineering-standards](https://github.com/amenses-innovation/engineering-standards) repository.
- Ensure you have access to the relevant GitHub team and repository.

## How to Contribute

### Reporting Bugs

Use the [Bug Report issue template](.github/ISSUE_TEMPLATE/bug_report.yml). Include reproduction steps, expected vs. actual behavior, and environment details.

### Suggesting Features

Use the [Feature Request issue template](.github/ISSUE_TEMPLATE/feature_request.yml). Describe the problem, proposed solution, and alternatives considered.

### Documentation

Use the [Documentation Request issue template](.github/ISSUE_TEMPLATE/documentation.yml) for docs improvements.

### Code Changes

1. **Find or create an issue** — link your PR to an issue when possible.
2. **Branch** — follow [docs/branching-strategy.md](docs/branching-strategy.md).
3. **Commit** — follow [docs/commit-guidelines.md](docs/commit-guidelines.md).
4. **Test** — add or update tests; ensure CI passes.
5. **Pull request** — select the correct [PR template](.github/PULL_REQUEST_TEMPLATE/).
6. **Review** — address feedback; maintainers will merge when approved.

## Pull Request Checklist

- [ ] Linked to a relevant issue (if applicable)
- [ ] Tests added or updated
- [ ] Documentation updated
- [ ] No secrets or credentials committed
- [ ] CI checks passing
- [ ] Appropriate reviewers assigned (see [CODEOWNERS](CODEOWNERS))

## Code Review Expectations

- Be respectful and constructive.
- Focus on correctness, security, maintainability, and clarity.
- Respond to review comments within two business days when possible.
